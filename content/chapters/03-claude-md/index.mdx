## CLAUDE.md란?

`CLAUDE.md`는 Claude Code에게 프로젝트의 규칙, 선호사항, 컨벤션을 알려주는 설정 파일입니다. 사람이 새 팀에 합류했을 때 온보딩 문서를 읽듯이, Claude도 이 파일을 읽고 프로젝트에 맞게 행동합니다.

<Callout type="info">
CLAUDE.md는 단순한 설정 파일이 아닙니다. **Claude의 행동을 결정하는 가장 중요한 파일**입니다. Boris Cherny는 팀 전체가 적극적으로 CLAUDE.md를 개선한다고 말합니다.
</Callout>

## 계층 구조

CLAUDE.md는 여러 위치에 존재할 수 있으며, 계층적으로 적용됩니다.

```
~/.claude/CLAUDE.md              ← 글로벌 (모든 프로젝트)
프로젝트루트/CLAUDE.md            ← 프로젝트 레벨
프로젝트루트/src/CLAUDE.md        ← 디렉토리 레벨
프로젝트루트/.claude/CLAUDE.md    ← 프로젝트 레벨 (git 공유)
```

하위 디렉토리의 설정이 상위를 오버라이드합니다. 팀과 공유할 설정은 `.claude/CLAUDE.md`에, 개인 설정은 `~/.claude/CLAUDE.md`에 저장하세요.

## 좋은 CLAUDE.md 작성법

### 기본 구조

```markdown title="CLAUDE.md 예시"
# 프로젝트 규칙

## 빌드 & 테스트
- 빌드: `pnpm build`
- 테스트: `pnpm test`
- 린트: `pnpm lint`

## 코딩 컨벤션
- TypeScript 사용
- 함수형 컴포넌트 + React hooks
- tailwindcss로 스타일링
- 한국어 주석 사용

## Git 규칙
- 커밋 메시지는 한글로 작성
- PR 제목 형식: [타입] 설명 (예: [feat] 로그인 기능 추가)

## 주의사항
- .env 파일을 절대 커밋하지 마세요
- API 키를 코드에 하드코딩하지 마세요
```

<Callout type="tip" title="프로 팁">
Claude가 실수를 했을 때, 다음에 같은 실수를 하지 않도록 CLAUDE.md에 규칙을 추가하세요. Boris Cherny 팀은 이 방식으로 Claude의 실수를 점점 줄여나갑니다.
</Callout>

## `#` 프리픽스로 메모리 저장

대화 중에 `#`으로 시작하는 메시지를 입력하면 Claude의 메모리에 바로 저장됩니다.

```
# Always use pnpm instead of npm
# 테스트는 vitest로 실행
# 컴포넌트는 항상 named export 사용
```

저장 위치를 한 번 지정하면 이후 자동으로 해당 CLAUDE.md에 추가됩니다. 파일을 직접 열지 않아도 대화 중에 바로 규칙을 추가할 수 있어 매우 편리합니다.

## `/init`으로 온보딩 문서 자동 생성

새 프로젝트를 시작할 때 `/init`을 실행하면 Claude가 코드베이스를 분석하여 CLAUDE.md를 자동 생성합니다.

```bash
/init
```

Claude는 다음을 분석합니다:
- 프로젝트 구조와 기술 스택
- package.json의 스크립트
- 테스트 프레임워크와 실행 방법
- 린팅/포매팅 설정
- 주요 디렉토리의 역할

<Callout type="tip">
`/init`으로 생성된 CLAUDE.md를 그대로 사용하지 말고, 팀의 실제 컨벤션에 맞게 보완하세요. 자동 생성된 문서는 좋은 시작점이지만, 팀만의 노하우를 추가해야 합니다.
</Callout>

## 팀과 공유하기

CLAUDE.md는 기본적으로 Git에 커밋하도록 설계되었습니다.

```bash title="팀과 CLAUDE.md 공유"
git add CLAUDE.md .claude/CLAUDE.md
git commit -m "Claude Code 프로젝트 규칙 추가"
```

팀 전체가 같은 규칙으로 Claude를 사용할 수 있어 코드 일관성이 크게 향상됩니다.
