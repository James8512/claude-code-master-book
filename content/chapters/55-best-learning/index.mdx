## 10억 토큰 규칙

Malcolm Gladwell의 "1만 시간의 법칙"을 들어보셨을 것입니다. 특정 분야에서 전문가가 되려면 1만 시간의 의도적 연습이 필요하다는 이론입니다. ykdojo는 AI 시대에 맞는 새로운 버전을 제안합니다.

<Callout type="tip" title="ykdojo의 10억 토큰 규칙">
"AI를 진정으로 이해하려면 많은 토큰을 소비하세요. 1만 시간이 아니라 10억 토큰입니다. 읽고, 쓰고, 실험하고, 실패하고, 다시 시도하세요. 그것이 AI와 함께 일하는 법을 배우는 유일한 방법입니다."
</Callout>

## 암벽 등반 비유

ykdojo는 암벽 등반에 비유하여 이 철학을 설명합니다.

> "암벽 등반을 잘하려면 암벽 등반을 하세요. 암벽 등반에 대한 책을 읽는 것이 아닙니다. 물론 기본적인 안전 지식과 기술은 필요하지만, 진짜 실력은 벽에 매달려 있을 때 만들어집니다."

이 원칙은 Claude Code 학습에도 그대로 적용됩니다.

| 학습 방법 | 효과 |
|-----------|------|
| 블로그 글 읽기 | 개념 이해에 도움, 실력 향상은 제한적 |
| 튜토리얼 따라하기 | 기본기 학습에 좋음, 실전과 거리 있음 |
| 실제 프로젝트에 적용 | 진짜 문제를 만나고 해결하며 실력 급성장 |
| 매일 사용하며 실험 | 패턴을 체화하고 직관이 발달 |

## 실천 중심 학습의 3단계

<Steps>
<Step title="일단 시작하라">
완벽한 준비를 기다리지 마세요. Claude Code를 설치하고, 가장 단순한 작업부터 시작하세요. "이 함수에 타입을 추가해줘" 같은 작은 요청이면 충분합니다.
</Step>
<Step title="매일 사용하라">
하루에 최소 하나의 작업을 Claude Code와 함께 하세요. 코드 리뷰, 리팩토링, 새 기능 구현, 버그 수정 등 어떤 작업이든 좋습니다. 꾸준함이 전문성을 만듭니다.
</Step>
<Step title="한계를 탐색하라">
익숙해지면 새로운 기능을 시도하세요. MCP 서버를 연결해보고, 커스텀 슬래시 커맨드를 만들어보고, CLAUDE.md를 최적화해보세요. 실패해도 괜찮습니다. 실패가 최고의 교사입니다.
</Step>
</Steps>

<Callout type="info" title="핵심 메시지">
이 책을 읽는 것만으로는 충분하지 않습니다. 책을 내려놓고 Claude Code를 열어 직접 시도해보세요. 한 챕터를 읽을 때마다 하나의 실습을 하는 것을 권장합니다.
</Callout>
