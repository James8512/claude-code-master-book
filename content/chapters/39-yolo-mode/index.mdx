## YOLO 모드란?

Claude Code는 파일 수정, 명령어 실행 등 위험할 수 있는 작업마다 사용자에게 승인을 요청합니다. 대부분의 경우 이것은 안전장치로서 중요하지만, 컨테이너 환경이나 반복 작업에서는 매번 승인하는 것이 비효율적입니다. 이때 사용하는 것이 바로 YOLO 모드입니다.

```bash
$ claude --dangerously-skip-permissions
```

이 플래그를 사용하면 Claude Code가 파일 편집, 셸 명령어 실행, MCP 도구 호출 등 모든 권한 요청을 자동으로 승인합니다.

<Callout type="warning" title="이름부터 경고입니다">
`--dangerously-skip-permissions`라는 이름 자체가 위험성을 알려줍니다. 이 모드는 Claude가 어떤 명령이든 확인 없이 실행할 수 있게 만듭니다. 반드시 안전한 환경에서만 사용하세요.
</Callout>

## 안전하게 사용할 수 있는 환경

<Steps>
<Step title="Docker 컨테이너 내부">
격리된 컨테이너에서는 호스트 시스템에 영향을 줄 수 없으므로 안전합니다. 컨테이너가 망가져도 다시 빌드하면 됩니다.

```bash
$ docker run -it my-dev-image claude --dangerously-skip-permissions
```
</Step>
<Step title="장시간 자율 작업">
대규모 리팩토링이나 코드 마이그레이션처럼 수백 개의 파일을 수정하는 작업에서, 매번 승인하는 대신 자율적으로 처리하게 할 수 있습니다.
</Step>
<Step title="신뢰할 수 있는 반복 작업">
이미 여러 번 수행하여 결과를 예측할 수 있는 작업, 예를 들어 일괄 포맷팅이나 린트 수정 같은 경우에 적합합니다.
</Step>
</Steps>

## 절대 사용하면 안 되는 환경

<Callout type="warning" title="금지 환경">
다음 환경에서는 YOLO 모드를 절대 사용하지 마세요:

- **호스트 시스템에서 직접 실행**: 시스템 파일이 삭제되거나 설정이 변경될 수 있습니다
- **중요한 데이터가 있는 디렉토리**: 복구할 수 없는 데이터 손실이 발생할 수 있습니다
- **프로덕션 서버**: 운영 중인 서비스에 치명적인 영향을 줄 수 있습니다
</Callout>

## 실전 활용: 컨테이너에서 대규모 마이그레이션

```bash
# Dockerfile에서 Claude Code 환경 구성
FROM node:20-slim
RUN npm install -g @anthropic-ai/claude-code

# 컨테이너 내부에서 자율 작업 실행
$ docker run -v ./project:/workspace -w /workspace my-claude-image \
  claude --dangerously-skip-permissions \
  -p "모든 JavaScript 파일을 TypeScript로 마이그레이션해줘"
```

컨테이너 안에서는 마음껏 실험할 수 있습니다. 결과가 마음에 들지 않으면 컨테이너를 버리고 처음부터 다시 시작하면 됩니다.

<Callout type="tip" title="핵심 원칙">
YOLO 모드의 핵심 원칙은 단순합니다: **망가져도 괜찮은 환경에서만 사용하세요.** 컨테이너는 일회용이지만, 여러분의 호스트 시스템은 그렇지 않습니다.
</Callout>
