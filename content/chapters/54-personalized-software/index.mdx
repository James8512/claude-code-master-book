## 개인화된 소프트웨어란

과거에는 소프트웨어를 만들려면 수주에서 수개월의 개발 시간이 필요했습니다. 하지만 AI 코딩 에이전트의 등장으로, 이제는 자신만의 필요에 맞는 도구를 1-2시간 안에 만들 수 있는 시대가 되었습니다. 이것이 바로 개인화된 소프트웨어의 시대입니다.

<Callout type="tip" title="ykdojo의 철학">
"원하는 것이 있으면 Claude Code에게 만들어달라고 하세요. 1-2시간 안에 완성됩니다. 시장에 나와 있는 도구가 80%만 맞다면, 나머지 20%를 직접 만드는 것이 이제는 현실적입니다."
</Callout>

## ykdojo의 개인화 사례들

### 1. 음성 전사 시스템

기존 음성 전사 도구들이 자신의 워크플로우에 완벽하게 맞지 않자, ykdojo는 Claude Code로 직접 만들었습니다.

- 키보드 단축키로 녹음 시작/종료
- 전사된 텍스트를 현재 커서 위치에 자동 삽입
- 자신의 작업 환경에 최적화된 UI

### 2. 시스템 프롬프트 패치

Claude Code의 시스템 프롬프트를 분석하고 자신의 워크플로우에 맞게 패치하는 도구를 직접 제작했습니다. 기존에는 불가능했던 커스터마이징을 소프트웨어로 자동화한 사례입니다.

### 3. Slack MCP CLI

Slack API와 MCP(Model Context Protocol)를 결합한 CLI 도구를 만들어, Claude Code에서 직접 Slack 메시지를 읽고 응답할 수 있게 했습니다. 터미널을 떠나지 않고도 팀 커뮤니케이션이 가능해졌습니다.

## 개인화 소프트웨어를 만드는 패턴

```
나는 매일 [특정 작업]을 반복하는데,
기존 도구 X는 [부족한 점]이 있어.
[원하는 기능]을 하는 CLI 도구를 만들어줘.
```

이 간단한 프롬프트 패턴으로 시작하면 됩니다. Claude Code는 여러분의 요구사항을 듣고, 설계하고, 구현하고, 테스트까지 도와줍니다.

## 언제 직접 만들어야 하는가

| 상황 | 판단 |
|------|------|
| 기존 도구가 90% 이상 맞을 때 | 기존 도구 사용 |
| 기존 도구가 70-90% 맞을 때 | 기존 도구 + 스크립트 보완 |
| 기존 도구가 70% 미만이거나 없을 때 | Claude Code로 직접 제작 |
| 매일 반복하는 수동 작업이 있을 때 | 반드시 자동화 도구 제작 |

<Callout type="info" title="새로운 사고방식">
"이런 도구가 있으면 좋겠다"에서 "이 도구를 만들자"로 사고방식이 바뀌는 것이 개인화 소프트웨어 시대의 핵심입니다. Claude Code는 그 간극을 메워주는 도구입니다.
</Callout>
