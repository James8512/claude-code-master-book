## 대화 기록의 가치

Claude Code와 나눈 대화에는 문제 해결 과정, 디버깅 히스토리, 아키텍처 결정 등 귀중한 지식이 담겨 있습니다. 이 기록을 효과적으로 검색하고 활용하면, 같은 문제를 다시 만났을 때 빠르게 해결할 수 있습니다.

## 대화 목록 확인

```bash
$ claude -r
```

`-r` 플래그를 사용하면 과거 대화 세션 목록을 확인할 수 있습니다. 각 세션의 시작 시간, 이름(있는 경우), 프로젝트 디렉토리가 표시됩니다.

## 특정 세션 재개

```bash
$ claude --resume session-name
```

이름이 붙은 세션은 `--resume` 플래그로 즉시 재개할 수 있습니다. 이전 컨텍스트가 그대로 유지되므로, 이어서 작업할 수 있습니다.

<Callout type="tip" title="세션 이름 붙이기">
중요한 작업을 할 때는 반드시 `/rename` 명령어로 세션에 의미 있는 이름을 붙이세요. "auth-refactor", "payment-bug-fix" 같은 이름이 나중에 찾을 때 큰 도움이 됩니다.
</Callout>

## 대화 내용 검색

특정 키워드가 포함된 과거 대화를 찾고 싶을 때는 대화 저장 디렉토리를 직접 검색할 수 있습니다.

```bash
# 대화 내용에서 특정 키워드 검색
$ grep -r "인증 미들웨어" ~/.claude/conversations/
```

이 방법으로 과거에 인증 미들웨어에 대해 논의한 모든 대화를 찾을 수 있습니다.

## 효과적인 기록 활용 전략

<Steps>
<Step title="프로젝트별 세션 이름 규칙">
"프로젝트명-작업내용" 형식으로 세션 이름을 통일하면 나중에 검색이 훨씬 쉬워집니다.

```
/rename masterbook-mdx-component
/rename api-server-auth-bug
/rename deploy-docker-optimization
```
</Step>
<Step title="주요 결정 사항 기록">
아키텍처 결정이나 중요한 디버깅 과정이 있었던 세션은 이름을 붙여두세요. 팀원에게 "왜 이렇게 결정했는지"를 설명할 때 해당 세션을 보여줄 수 있습니다.
</Step>
<Step title="정기적인 정리">
오래된 세션 중 더 이상 필요 없는 것은 정리하고, 중요한 세션만 이름을 붙여 보관합니다.
</Step>
</Steps>

<Callout type="info" title="대화 기록은 팀 자산">
개인의 대화 기록도 가치 있지만, `/export`로 내보낸 대화는 팀의 지식 자산이 됩니다. 복잡한 버그를 해결한 과정을 내보내 팀 위키에 기록해두면, 비슷한 문제가 발생했을 때 팀 전체가 혜택을 봅니다.
</Callout>
