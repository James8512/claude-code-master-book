## 커스텀 커맨드란?

자주 쓰는 프롬프트를 마크다운 파일로 저장하면, `/명령어` 형태로 바로 실행할 수 있습니다. 같은 프롬프트를 반복 입력할 필요가 없어집니다.

## 커맨드 만들기

`.claude/commands/` 디렉토리에 마크다운 파일을 생성합니다.

```markdown title=".claude/commands/explain.md"
다음 파일을 분석하고 설명해줘:
- 파일의 목적과 역할
- 주요 함수/클래스의 동작
- 다른 모듈과의 의존 관계
- 개선할 수 있는 부분

대상 파일: $ARGUMENTS
```

이제 `/explain src/auth.ts`처럼 사용할 수 있습니다. `$ARGUMENTS`는 슬래시 커맨드 뒤에 입력한 텍스트로 대체됩니다.

## 실용적인 커맨드 예시

### 코드 리뷰 커맨드

```markdown title=".claude/commands/review.md"
다음 파일의 코드를 리뷰해줘:
1. 버그 가능성이 있는 부분
2. 성능 개선 포인트
3. 보안 취약점
4. 코딩 컨벤션 위반

심각도(높음/중간/낮음)와 함께 구체적인 수정 제안을 해줘.

대상: $ARGUMENTS
```

### 테스트 생성 커맨드

```markdown title=".claude/commands/test.md"
$ARGUMENTS에 대한 테스트를 작성해줘.

요구사항:
- vitest 사용
- 성공/실패/엣지 케이스 포함
- describe/it 구조
- 각 테스트에 한글 설명

기존 테스트 패턴을 따라서 작성해줘.
```

### 리팩토링 커맨드

```markdown title=".claude/commands/refactor.md"
$ARGUMENTS를 리팩토링해줘.

원칙:
- 기능 변경 없이 구조만 개선
- 함수는 단일 책임 원칙 준수
- 중복 코드 제거
- 변경 전후 동작이 동일함을 보장
```

## 팀과 공유하기

커맨드 파일을 Git에 커밋하면 팀 전체가 동일한 커맨드를 사용할 수 있습니다.

```bash
git add .claude/commands/
git commit -m "커스텀 Claude Code 커맨드 추가"
```

<Callout type="tip" title="Boris Cherny의 팁">
