## 가르치면서 배운다

ykdojo는 Claude Code 팁을 공유하기 시작하면서 오히려 더 많이 배웠다고 말합니다. 지식 공유는 일방통행이 아니라 양방향 도로입니다. 누군가에게 설명하기 위해 깊이 이해해야 하고, 다른 사람들의 피드백에서 새로운 인사이트를 얻게 됩니다.

<Callout type="tip" title="ykdojo의 경험">
"지식 공유는 일방통행이 아닙니다. 팁을 공유할 때마다 댓글에서 새로운 것을 배웠습니다. 커뮤니티는 개인보다 항상 더 많이 알고 있습니다."
</Callout>

## 공유에서 발견한 것들

### --system-prompt 플래그 발견

ykdojo가 Claude Code 팁을 공유하던 중, Reddit 댓글에서 `--system-prompt` 플래그의 존재를 알게 되었습니다. 이 플래그를 통해 Claude Code의 시스템 프롬프트를 직접 확인할 수 있었고, 이것이 더 깊은 이해와 새로운 활용법의 발견으로 이어졌습니다.

### 새로운 사용 사례 발견

Reddit과 커뮤니티에서 다른 사용자들이 공유하는 사례를 통해, 자신이 생각하지 못했던 완전히 새로운 활용 방법들을 발견했습니다. 지식 공유의 생태계에 참여하는 것 자체가 학습의 가속기가 됩니다.

## Anthropic에 직접 기여하기

ykdojo는 단순히 팁을 공유하는 것에서 나아가, Claude Code 자체에도 기여했습니다.

<Steps>
<Step title="/permissions 검색 기능 기여">
Claude Code의 `/permissions` 명령어에 검색 기능을 추가하는 기여를 했습니다. 권한 목록이 길어질수록 필요한 권한을 찾기 어려웠는데, 검색 기능으로 이 문제를 해결했습니다.
</Step>
<Step title="스크롤 위치 수정 기여">
대화가 길어질 때 스크롤 위치가 예상과 다르게 동작하는 버그를 발견하고 수정하여 기여했습니다. 실제 사용자로서 발견한 문제를 직접 해결한 사례입니다.
</Step>
</Steps>

## 지식 공유를 시작하는 방법

| 방법 | 난이도 | 영향력 |
|------|--------|--------|
| 사내 Slack에 팁 공유 | 낮음 | 팀 내 |
| 블로그 글 작성 | 중간 | 커뮤니티 |
| Reddit/커뮤니티 참여 | 낮음 | 커뮤니티 |
| 오픈소스 기여 | 높음 | 전체 사용자 |
| 발표/강의 | 높음 | 폭넓음 |

<Callout type="info" title="시작하기">
거창할 필요 없습니다. 오늘 배운 작은 팁 하나를 팀 채널에 공유하는 것으로 시작하세요. 그 작은 시작이 선순환의 첫 걸음입니다.
</Callout>
