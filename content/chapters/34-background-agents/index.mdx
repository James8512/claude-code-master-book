## 개요

Claude Code에서 작업 중 다른 일을 해야 할 때, 현재 작업을 백그라운드로 보내고 새로운 작업을 시작할 수 있습니다. 또한 서브에이전트를 백그라운드에서 실행하여 여러 모듈을 동시에 처리하는 것도 가능합니다.

## Ctrl+B: 백그라운드로 전환

현재 실행 중인 Claude Code 작업을 백그라운드로 보내는 단축키입니다:

<KeyCombo keys={["Ctrl", "B"]} />

```
# 시나리오: 대규모 리팩토링 작업 중
> 모든 API 엔드포인트에 에러 핸들링을 추가해줘

# 작업이 진행되는 동안 Ctrl+B를 눌러 백그라운드로 전환
# 새로운 프롬프트 입력 가능

> 백그라운드 작업 상태를 확인해줘
```

## BashOutput 도구로 상태 확인

백그라운드에서 실행 중인 작업의 상태를 확인할 수 있습니다:

```
백그라운드에서 실행 중인 작업의 진행 상황을 알려줘.
```

Claude는 BashOutput 도구를 사용하여 백그라운드 프로세스의 출력을 읽고 현재 상태를 보고합니다.

## 서브에이전트 백그라운드 실행

여러 모듈을 동시에 작업하는 병렬 패턴:

```
다음 모듈들을 서브에이전트로 동시에 처리해줘:

1. src/auth/ - 인증 모듈 리팩토링
2. src/payment/ - 결제 모듈 테스트 추가
3. src/notification/ - 알림 모듈 타입 정의 추가

각 서브에이전트는 독립적으로 작업하고,
모두 완료되면 결과를 종합해줘.
```

<Steps>
<Step title="작업 분할">
독립적으로 처리 가능한 모듈을 식별합니다. 서로 의존성이 없는 작업이 이상적입니다.
</Step>
<Step title="병렬 서브에이전트 생성">
각 모듈에 대한 서브에이전트를 생성하고, 전문화된 지시를 부여합니다.
</Step>
<Step title="병렬 실행">
서브에이전트들이 동시에 작업을 수행합니다. 각각 독립된 컨텍스트에서 실행됩니다.
</Step>
<Step title="결과 종합">
모든 서브에이전트의 작업이 완료되면, 메인 에이전트가 결과를 수집하고 통합합니다.
</Step>
</Steps>

<Callout type="tip" title="병렬 작업의 핵심 조건">
서브에이전트의 병렬 실행이 효과적이려면 **각 작업이 독립적**이어야 합니다. 서로 같은 파일을 수정하는 작업을 병렬로 실행하면 충돌이 발생합니다.
</Callout>

<Callout type="warning" title="리소스 관리">
백그라운드 에이전트는 메모리와 토큰을 계속 소비합니다. 불필요한 백그라운드 작업은 종료하여 리소스를 확보하세요.
</Callout>
