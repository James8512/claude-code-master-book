## 코드를 아는 AI가 쓰는 문서

Claude Code의 글쓰기 보조 기능이 특별한 이유는, 단순히 글을 잘 쓰는 것이 아니라 **코드를 이해한 상태에서** 문서를 작성한다는 점입니다. 실제 구현과 동떨어진 문서가 아니라, 코드와 일치하는 정확한 문서를 만듭니다.

## 기술 문서 작성: OpenAPI 스펙

```
> 이 프로젝트의 모든 API 엔드포인트를 분석해서
  OpenAPI 3.0 스펙 문서를 YAML로 작성해줘.
  각 엔드포인트의 요청/응답 스키마, 에러 코드, 인증 방식을 포함해줘
```

Claude Code는 실제 라우터 코드, 미들웨어, 타입 정의를 분석하여 정확한 API 문서를 생성합니다. 수동으로 작성하면 몇 시간이 걸리는 작업을 몇 분 만에 완료합니다.

<Callout type="info" title="코드와 동기화되는 문서">
코드가 변경되면 Claude Code에게 "API 스펙을 최신 코드에 맞게 업데이트해줘"라고 요청하면 됩니다. 코드를 직접 분석하므로 문서와 구현의 불일치가 발생하지 않습니다.
</Callout>

## 블로그 포스트 작성

이미 구현한 코드를 기반으로 기술 블로그 포스트를 작성할 수 있습니다.

```
> 방금 구현한 실시간 알림 시스템의 기술적 과정을 블로그 포스트로 작성해줘.
  - 문제 상황과 요구사항
  - 기술 선택의 이유 (WebSocket vs SSE)
  - 핵심 구현 코드 (주요 부분만 발췌)
  - 성능 측정 결과
  - 배운 점과 개선할 점
```

Claude Code는 실제로 코드를 작성한 과정의 컨텍스트를 가지고 있으므로, "왜 이런 선택을 했는지"까지 자연스럽게 설명할 수 있습니다.

## 글쓰기 팁

<Callout type="warning" title="ykdojo의 조언">
"마지막 문단에서 이전 내용을 반복하지 마." ykdojo는 AI가 글을 쓸 때 흔히 보이는 패턴을 지적합니다. 결론 부분에서 "이번 글에서는 A, B, C를 살펴보았습니다"와 같은 반복은 독자의 시간을 낭비합니다. Claude Code에게 이런 패턴을 피하도록 명시적으로 지시하세요.
</Callout>

### 효과적인 글쓰기 프롬프트

```
> 블로그 포스트를 작성할 때 다음 규칙을 지켜줘:
  - 마지막 문단에서 이전 내용을 반복하지 마
  - 불필요한 서론 없이 바로 본론으로 들어가
  - 코드 예제는 실제 동작하는 코드만 사용해
  - 독자가 바로 따라할 수 있는 실용적인 내용 위주로 작성해
```

## README 자동 생성

```
> 이 프로젝트의 README.md를 작성해줘.
  실제 package.json, 설정 파일, 디렉토리 구조를 분석해서
  설치 방법, 사용법, 환경 변수 설정을 정확하게 작성해줘
```

Claude Code가 프로젝트 구조를 실제로 분석하므로, "가상의 예제"가 아닌 실제 프로젝트에 맞는 정확한 문서가 생성됩니다.
