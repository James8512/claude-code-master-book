## 세션 이어가기

### `--continue` — 직전 대화 이어가기
```bash
claude --continue
```
터미널을 닫거나 노트북이 꺼져도 직전 대화를 바로 이어서 작업합니다.

### `--resume` — 세션 선택하여 이어가기
```bash
claude --resume
```
지난 세션 목록에서 골라 재개합니다. 이전 맥락을 다시 설명할 필요가 없습니다.

### `/rename` — 세션 이름 붙이기
```bash
/rename api-migration
```
세션에 이름을 붙이면 나중에 바로 찾을 수 있습니다.

```bash
claude --resume api-migration
```

## 원격 세션

### `&`로 백그라운드 실행
프롬프트 앞에 `&`를 붙이면 작업을 Claude Code 웹으로 보내 백그라운드에서 실행합니다.

```
& 이 프로젝트의 모든 TODO를 찾아서 이슈로 정리해줘
```

### `--teleport`로 로컬에 불러오기
외부에서 시작한 작업을 로컬 터미널에 가져옵니다.

```bash
claude --teleport 세션ID
```

<Callout type="tip">
Boris Cherny는 로컬 5개 + 웹 10개 = 총 15개 세션을 동시에 활용합니다. `&`로 백그라운드 실행하고, `--teleport`로 필요할 때 로컬에 불러오는 방식입니다.
</Callout>

## `-p` Headless 모드

대화형 UI 없이 명령어처럼 실행합니다.

```bash
claude -p "이 프로젝트의 의존성을 분석해줘"
```

CI/CD 파이프라인이나 스크립트에 통합할 때 유용합니다.

```bash title="Git 훅에서 사용"
# .git/hooks/pre-commit
claude -p "스테이징된 파일에 보안 취약점이 있는지 확인해줘"
```

## 워크플로우 패턴

### 아침 루틴
```bash
claude --continue     # 어제 하던 작업 이어가기
/stats               # 사용 현황 확인
```

### 병렬 작업
```bash
# 터미널 1: 기능 구현
claude

# 터미널 2: 테스트 작성
claude

# 터미널 3: 백그라운드 작업
& 문서 업데이트해줘
```
