## 워크플로우에 투자하라

ykdojo는 Claude Code를 단순한 코딩 도구가 아닌, 자신만의 워크플로우를 구축하는 플랫폼으로 활용합니다. 반복되는 작업을 자동화하면 시간이 지날수록 생산성이 기하급수적으로 증가합니다.

## ykdojo의 커스텀 워크플로우 사례

### 음성 전사 시스템

ykdojo는 키보드 단축키를 누르면 음성을 녹음하고, 자동으로 전사하여 커서 위치에 텍스트를 삽입하는 시스템을 Claude Code로 구축했습니다. 코딩 중에 손을 키보드에서 떼지 않고도 메모를 남길 수 있습니다.

### 커스텀 상태 표시줄

터미널 하단에 현재 프로젝트 상태, Git 브랜치, 최근 테스트 결과 등을 실시간으로 표시하는 상태 표시줄을 구축했습니다.

### /commit 슬래시 커맨드

커밋 메시지를 자동으로 생성하는 커스텀 슬래시 커맨드입니다. 변경 사항을 분석하고 컨벤셔널 커밋 형식의 메시지를 자동으로 작성합니다.

### /gha 슬래시 커맨드

GitHub Actions 워크플로우를 디버깅하는 전용 커맨드입니다. 실패한 워크플로우 로그를 자동으로 가져와서 분석합니다.

### /handoff 슬래시 커맨드

현재 작업 상태를 요약하여 새로운 세션에 전달하는 커맨드입니다. 컨텍스트 윈도우가 가득 찼을 때 작업을 이어가는 데 사용합니다.

<Callout type="info" title="슬래시 커맨드 만들기">
`.claude/commands/` 디렉토리에 마크다운 파일을 생성하면 커스텀 슬래시 커맨드가 됩니다. 예를 들어 `.claude/commands/commit.md` 파일을 만들면 `/commit`으로 실행할 수 있습니다.
</Callout>

## 나만의 워크플로우 구축하기

<Steps>
<Step title="반복 작업 식별">
일주일 동안 자신의 작업 패턴을 관찰하세요. 매일 또는 매주 반복하는 작업이 무엇인지 기록합니다.

- 매번 같은 형식의 커밋 메시지를 작성하는가?
- PR 리뷰 때마다 같은 체크리스트를 확인하는가?
- 배포 전에 항상 같은 절차를 거치는가?
</Step>
<Step title="자동화 설계">
식별한 반복 작업을 어떻게 자동화할 수 있을지 설계합니다. 슬래시 커맨드, 셸 스크립트, 또는 두 가지의 조합이 될 수 있습니다.
</Step>
<Step title="Claude Code로 구현">
```
> .claude/commands/deploy-check.md 파일을 만들어줘.
  배포 전 체크리스트를 자동으로 실행하는 커맨드야.
  1. 모든 테스트 실행
  2. 린트 검사
  3. 타입 검사
  4. 빌드 확인
  5. 환경 변수 검증
```
</Step>
<Step title="테스트 및 개선">
만든 워크플로우를 실제로 사용해보고, 불편한 점을 개선합니다. 한 번에 완벽할 필요는 없습니다. 점진적으로 발전시키세요.
</Step>
<Step title="팀과 공유">
유용한 워크플로우는 `.claude/commands/` 디렉토리를 Git에 커밋하여 팀원과 공유합니다. 팀 전체의 생산성이 향상됩니다.
</Step>
</Steps>

<Callout type="tip" title="작은 것부터 시작하세요">
처음부터 거창한 워크플로우를 만들려고 하지 마세요. 가장 자주 반복하는 단 하나의 작업을 자동화하는 것부터 시작하세요. 그 작은 성공이 더 큰 자동화로 이어집니다.
</Callout>
