## 토큰 비용 이해하기

Claude Code는 사용한 토큰량에 따라 구독 한도가 소모됩니다. 효율적인 토큰 관리는 더 많은 작업을 수행할 수 있게 해줍니다.

## `/context`로 토큰 사용 확인

```bash
/context
```

시스템 프롬프트, MCP 서버, 메모리 파일 등 무엇이 컨텍스트를 차지하는지 확인합니다. 불필요한 요소를 정리하면 토큰을 절약할 수 있습니다.

## `/usage`로 한도 관리

```bash
/usage
```

현재 토큰 사용량과 한도를 확인합니다. `/extra-usage`로 제한을 해제할 수도 있습니다.

## 모델 선택 전략

| 모델 | 특징 | 사용 시점 |
|------|------|-----------|
| Claude Opus | 가장 강력, 토큰 소모 높음 | 복잡한 아키텍처, 큰 리팩토링 |
| Claude Sonnet | 균형 잡힌 성능 | 일상적인 코딩 작업 |
| Claude Haiku | 빠르고 경제적 | 간단한 질문, 빠른 수정 |

<Callout type="tip" title="Boris Cherny의 조언">
Boris는 항상 Opus를 사용합니다. 큰 모델이 결국 더 빠릅니다. 도구 사용을 잘하고 사람의 개입이 적어도 되기 때문에, 작은 모델보다 결과적으로 효율적입니다.
</Callout>

## 토큰 절약 팁

### 1. `/compact`로 대화 요약
대화가 길어지면 `/compact`를 실행하여 핵심만 유지합니다.

### 2. `!` 프리픽스로 직접 실행
`!git status`처럼 모델 추론 없이 명령을 바로 실행합니다.

### 3. 구체적인 프롬프트 작성
모호한 프롬프트는 여러 번의 왕복을 필요로 합니다. 한 번에 정확히 지시하세요.

### 4. `/clear`로 새 시작
주제가 바뀌면 `/clear`로 불필요한 맥락을 제거하세요.

### 5. CLAUDE.md 최적화
CLAUDE.md가 너무 길면 매 대화마다 토큰을 소모합니다. 핵심만 간결하게 유지하세요.
