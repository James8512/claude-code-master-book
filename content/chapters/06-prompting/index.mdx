Claude Code에서 좋은 프롬프트란 **명확하고 구체적이며 맥락을 제공하는** 프롬프트입니다.

### 나쁜 프롬프트의 특징
- 너무 모호함: "코드 좀 개선해줘"
- 맥락 부족: "버그 고쳐줘"
- 범위가 너무 넓음: "전체 리팩토링해줘"

### 좋은 프롬프트의 특징
- 구체적인 파일/함수 지정
- 현재 상태와 원하는 결과 설명
- 제약 조건 명시

## Extended Thinking

API에서 Extended Thinking을 활성화하면 Claude가 답변 전에 추론 과정을 더 길게 가져갑니다.

```typescript title="Extended Thinking 설정 (API)"
const response = await anthropic.messages.create({
  model: "claude-opus-4-5-20250929",
  max_tokens: 16000,
  thinking: {
    type: "enabled",
    budget_tokens: 10000
  },
  messages: [{ role: "user", content: "복잡한 질문..." }]
});
```

설정한 토큰 예산 안에서 "thinking blocks"로 먼저 추론하고, 그다음 최종 답변을 출력합니다. 복잡한 아키텍처 설계나 까다로운 버그 분석에 특히 효과적입니다.

<Callout type="info">
Extended Thinking은 Claude API에서 사용하는 기능입니다. Claude Code에서는 자체적으로 사고 과정을 관리하므로 별도 설정 없이도 복잡한 작업을 처리합니다.
</Callout>

## 프롬프트 패턴 모음

### 버그 수정
```

수량은 항상 1 이상이어야 해. 0 이하로 내려가면 아이템을 제거해줘.
```

