## Boris Cherny의 13가지 프로 셋업

Claude Code의 창시자 Boris Cherny는 주당 PR 100개를 혼자 처리합니다. 그의 13가지 셋업을 하나씩 살펴봅니다.

### 1. 15개 세션 병렬 실행
로컬 5개 + 웹 10개 세션을 동시에 운영합니다. `&`로 웹 세션을 시작하고 `--teleport`로 로컬에 불러옵니다. 사용자 인풋이 필요할 때 시스템 알림이 오도록 설정합니다.

### 2. 항상 Opus 사용
작은 모델이 빠르다는 환상을 버리세요. Opus가 도구 사용을 잘하고 사람의 개입이 적어 결국 더 빠른 결과를 냅니다.

### 3. CLAUDE.md를 팀과 공유
CLAUDE.md는 Git 커밋하도록 설계된 파일입니다. Claude의 실수를 기록하여 반복을 방지하고, 팀원 모두가 적극적으로 개선합니다.

### 4. PR 리뷰에 Claude 태그
`/install-github-app`으로 GitHub에 Claude를 설치하고, PR 리뷰 시 Claude를 태그하여 CLAUDE.md에 규칙을 추가합니다.

### 5. Plan 모드에서 시작
대부분의 세션을 Plan 모드로 시작합니다. 계획을 여러 번 수정하고, 마음에 드는 플랜이 나왔을 때 Auto-accept 모드로 실행하면 보통 한 번에 원하는 결과가 나옵니다.

### 6. 워크플로우를 커맨드로 저장
반복 프롬프트를 `.claude/commands/`에 저장하고 Git으로 팀과 공유합니다. Claude가 bash를 실행할 수 있으므로 자동화 범위가 넓습니다.

### 7. 서브에이전트 규칙적 사용
code simplifier 서브에이전트로 완료된 코드를 정리하고, verify 에이전트로 테스트를 검증합니다.

### 8. Post-tool 훅으로 포매팅
Claude가 파일을 수정할 때마다 자동으로 포매팅합니다. 나머지 10%의 포매팅 문제를 자동 해결합니다.

### 9. YOLO 대신 /permissions
`/permissions`로 필요한 명령을 사전 등록하고, `settings.json`에 저장하여 팀과 공유합니다.

### 10. MCP로 모든 도구 연결
Slack 검색, BigQuery 쿼리, Sentry 에러 로그 등 모든 도구에 접근 권한을 부여합니다.

### 11. 장기 작업 핸들링
백그라운드 에이전트로 작업을 검증하고, Agent Stop 훅으로 디터미니스틱한 결과를 보장하며, 무한 이터레이션 플러그인을 활용합니다.

### 12. 자기 검증 방법 제공
Claude에게 작업 검증 방법을 정확히 제시합니다. 도메인별로 브라우저 테스트, 시뮬레이터 테스트, bash 명령 등 적합한 검증 방식을 사용합니다.

### 13. 큰 모델이 결국 빠르다
작은 모델로 아끼려다 결국 더 많은 시간과 수정이 필요합니다. 처음부터 가장 강력한 모델을 사용하세요.

## 실전 패턴

### 디버깅 패턴
```
이 에러를 디버깅해줘: [에러 메시지]
1. 에러의 근본 원인을 분석해
2. 관련 코드를 찾아
3. 수정 방안을 제안하고 적용해
4. 수정 후 테스트가 통과하는지 확인해
```

### 리팩토링 패턴
```
이 모듈을 리팩토링해줘:
1. 현재 구조를 분석해
2. 개선 포인트를 목록으로 정리해
3. 기능 변경 없이 구조만 개선해
4. 테스트가 여전히 통과하는지 확인해
```

### 코드 리뷰 패턴
```
이 PR의 코드를 리뷰해줘:
- 버그 가능성
- 성능 이슈
- 보안 취약점
- 컨벤션 위반
심각도와 함께 구체적인 수정 제안을 해줘.
```

<Callout type="tip">
Claude에게 자기 검증을 시키면 2~3배 더 좋은 결과를 얻을 수 있습니다. "수정한 후 테스트가 통과하는지 확인해줘"를 항상 추가하세요.
</Callout>
