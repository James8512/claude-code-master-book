## 개요

대규모 데이터 처리, 빌드, 테스트 등 장시간 실행되는 작업을 Claude Code에게 맡긴 후, 1분마다 확인하면 불필요한 토큰이 소비됩니다. 수동 지수 백오프 전략은 확인 간격을 점진적으로 늘려 토큰을 절약하면서도 작업 완료를 놓치지 않는 방법입니다.

## 지수 백오프 패턴

확인 간격을 2배씩 늘려갑니다:

| 확인 횟수 | 대기 시간 | 누적 시간 |
|----------|-----------|-----------|
| 1회차 | 1분 | 1분 |
| 2회차 | 2분 | 3분 |
| 3회차 | 4분 | 7분 |
| 4회차 | 8분 | 15분 |
| 5회차 | 16분 | 31분 |

## 활용 예시: 대용량 CSV 처리

```
10GB CSV 파일을 파싱해서 데이터베이스에 적재해줘.
진행 상황은 내가 물어볼 때만 알려줘.
중간에 에러가 발생하면 로그 파일에 기록하고 계속 진행해줘.
처리 완료되면 요약 통계를 보여줘.
```

<Callout type="tip" title="핵심 원칙">
Claude에게 작업을 맡긴 후 **"진행 상황은 내가 물어볼 때만 알려줘"**라고 명시하세요. 그러면 Claude가 자발적으로 상태를 보고하지 않아 토큰이 절약됩니다.
</Callout>

## 병렬 작업과의 조합

지수 백오프의 진정한 가치는 대기 시간을 다른 작업에 활용할 수 있다는 점입니다:

<Steps>
<Step title="장시간 작업 시작">
첫 번째 터미널에서 대규모 작업을 시작합니다.
</Step>
<Step title="1분 후 확인, 다른 작업 전환">
1분 후 진행 상황을 확인하고, 정상이면 두 번째 터미널에서 다른 작업을 시작합니다.
</Step>
<Step title="2분 후 확인, 작업 계속">
2분 후 다시 확인하고, 두 번째 작업을 계속합니다.
</Step>
<Step title="4분 후 확인">
이후 4분, 8분 간격으로 확인하며 대부분의 시간은 다른 작업에 집중합니다.
</Step>
</Steps>

<Callout type="info" title="토큰 절약 효과">
1분마다 확인하면 30분간 30번의 확인이 필요합니다. 지수 백오프를 사용하면 같은 30분간 5번만 확인하면 됩니다. 확인 횟수 83% 감소, 그만큼 토큰과 비용이 절약됩니다.
</Callout>
