## 서브에이전트란?

서브에이전트는 각각 200k 컨텍스트를 가진 보조 에이전트입니다. 특정 작업을 맡아 동시에 처리한 뒤 결과를 메인 작업으로 합칩니다.

조사, 정리, 분석처럼 역할을 나눠 위임하면 한 번의 요청으로 더 많은 일을 효율적으로 처리할 수 있습니다.

## 서브에이전트 활용 패턴

### 코드 검증 에이전트

```
코드를 작성한 후 서브에이전트로 코드 품질을 검증해줘.
- 타입 안전성 확인
- 에지 케이스 검토
- 성능 이슈 체크
```

### 코드 단순화 에이전트

```
작업이 완료되면 code simplifier 서브에이전트로 코드를 정리해줘.
- 불필요한 복잡성 제거
- 중복 코드 통합
- 변수명 개선
```

<Callout type="tip" title="Boris Cherny의 팁">
Boris는 code simplifier 서브에이전트와 verify 에이전트를 규칙적으로 사용합니다. 커스텀 서브에이전트는 항상 사용하는 기능을 자동화하는 목적으로 만드세요.
</Callout>

## Agent Skills

Agent Skills는 특정 작업 방법을 폴더 단위로 묶어 Claude에게 가르치는 방식입니다. 한 번 만들어두면 다른 프로젝트에서도 재사용할 수 있습니다.

### Skills 디렉토리 구조

```
.claude/skills/
├── code-review/
│   ├── instructions.md
│   └── examples/
├── testing/
│   ├── instructions.md
│   └── templates/
└── deployment/
    └── instructions.md
```

## 플러그인

플러그인은 명령어, 에이전트, 스킬, 훅, MCP 설정을 하나로 묶어 배포하는 방법입니다.

```bash title="플러그인 설치"
/plugin install my-setup
```

여러 파일과 설정을 따로 주고받지 않아도 한 줄로 작업 환경을 설치할 수 있어, 팀 워크플로를 빠르게 재현할 수 있습니다.
