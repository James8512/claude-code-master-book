## Chrome 통합이란?

Claude Code는 Chrome 브라우저와 네이티브로 통합되어, 웹 페이지를 직접 탐색하고 조작할 수 있습니다. 코드와 브라우저를 오가며 작업하는 프론트엔드 개발에서 특히 강력한 기능입니다.

## 설치 및 시작

<Steps>
<Step title="Chrome 확장 프로그램 설치">
claude.ai/chrome에서 공식 Chrome 확장 프로그램을 설치합니다. 이 확장은 Claude Code와 브라우저 사이의 통신을 담당합니다.
</Step>
<Step title="Chrome 모드로 실행">
```bash
$ claude --chrome
```
이 플래그로 실행하면 Claude Code가 Chrome 브라우저에 연결됩니다.
</Step>
</Steps>

## 주요 기능

### 페이지 탐색

Claude Code가 직접 URL로 이동하여 페이지 내용을 확인할 수 있습니다.

```
> localhost:3000의 메인 페이지를 열어서 레이아웃을 확인해줘
```

### 버튼 클릭과 폼 입력

실제 사용자처럼 버튼을 클릭하고 폼을 채울 수 있습니다.

```
> 로그인 페이지에서 이메일에 "test@example.com",
  비밀번호에 "password123"을 입력하고 로그인 버튼을 클릭해줘
```

### 콘솔 에러 읽기

브라우저 콘솔에 출력된 에러 메시지를 직접 확인하고 원인을 분석합니다.

```
> 이 페이지에서 콘솔 에러가 있는지 확인해줘
```

### DOM 검사

페이지의 DOM 구조를 직접 검사하여 요소의 속성, 스타일, 구조를 파악합니다.

```
> 네비게이션 바의 DOM 구조를 확인하고 접근성 속성이 올바른지 점검해줘
```

### 스크린샷 캡처

현재 페이지의 스크린샷을 찍어 시각적으로 레이아웃을 확인할 수 있습니다.

```
> 현재 페이지의 스크린샷을 찍어서 모바일 뷰에서 레이아웃이 깨지는지 확인해줘
```

## 실전 사례: 버그 재현부터 수정까지

프론트엔드 버그를 보고받았을 때, Chrome 통합을 활용하면 재현부터 수정까지 한 번에 처리할 수 있습니다.

<Steps>
<Step title="버그 재현">
```
> localhost:3000/products에서 '장바구니 담기' 버튼을 클릭했을 때
  콘솔에 에러가 발생하는지 확인해줘
```
Claude가 페이지를 열고, 버튼을 클릭하며, 콘솔 에러를 캡처합니다.
</Step>
<Step title="원인 분석">
```
> 콘솔 에러 메시지를 기반으로 관련 소스 코드를 찾아서 원인을 분석해줘
```
에러 메시지에서 파일명과 라인 번호를 추출하여 코드를 분석합니다.
</Step>
<Step title="코드 수정">
```
> 원인을 찾았으면 수정해줘
```
코드를 수정하고 저장합니다.
</Step>
<Step title="수정 확인">
```
> 다시 같은 동작을 반복해서 에러가 사라졌는지 확인해줘
```
수정 후 동일한 동작을 반복하여 버그가 해결되었는지 검증합니다.
</Step>
</Steps>

<Callout type="tip" title="프론트엔드 개발의 새로운 패러다임">
Chrome 통합을 사용하면 "코드 수정 → 브라우저 전환 → 새로고침 → 확인 → 코드 에디터 전환"의 반복 루프를 Claude Code가 자동으로 처리합니다. 개발자는 지시만 하면 됩니다.
</Callout>
